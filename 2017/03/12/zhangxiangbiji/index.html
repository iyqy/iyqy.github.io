2016级java组张祥笔记

  • 组员笔记

输入/输出

1.流概述

  • 输入输出流（IO）：程序通道
  • 源
  • 目的地
    2.输入/输出流
  • 所有输入流都是抽象类InputStream(字节输入流)或者抽象类Reader(字符输入流)的子类
  • 所有输出流都是抽象类OutputStream（字节输出流）或抽象类Writer（字符输入流）的子类
  • System.in与System.out分别为InputStream与PrintStream的静态对象
    3.实例输入/输出对象时,将输入/输出流作为参数构造另外一个流的对象

4.输入

    • InputStream
    • 构造方法(InputStream)
    • read():从源中获取数据传入到流中
    • DataInputStream
      -
  • -ZipInputStream
    -
    • (字符处理)Reader
    • BufferedReader:数据输入流
    1. 构造方法(Writer流对象)
    2. readLine();获取流中的一行，返回值为字符串
    • FileReader:文件数据输入流
      1.构造方法(文件)
  1. 输出
    • OutputStream
    • 构造方法(OutputStream)
    • write()将数据转入流中，参数为字符相关
    • flush()将流中的数据转入目的地
    • DataOutputStream

    • FileWriter: 文件字符输出流
    1. 构造方法(文件)

多线程

  • 并发执行的机制称为多线程
  1. 继承java.lang.Thread类
  • ==该子类应重写 Thread 类的 run 方法。==
  • 继承了Thread类的类重写public void run()方法,利用Thread中的start()运行该线程
  1. 实现接口java.lang.Runnable
  • ①建立Runnable对象
  • ②使用参数为Runnable对象的构造方法创造Thread实例
  • ③调用start()方法启动线程
  1. 线程的生命周期
  • 出生：Thread t=new Thread();
  • 就绪: t.start();或者时间片结束；
    • sleep();
    • wait();
    • 等待输入/输出完成
    • notify();
    • notifyAll();
    • interrupt();
    • 休眠时间结束
    • 输入/输出结束
  • 执行: 得到系统资源时
    • 等待:t.wait(); t.notifyAll()或者T.notify()唤醒进程
      • 休眠：sleep()，进入休眠状态
      • 阻塞：运行时发出输入/输出请求时
      • 死亡：run()执行完毕
  1. 线程的操作
  • 线程的休眠sleep();在run方法中使用Thrad.sleep(time/ms);需要捕获异常
  • 线程的加入:Thread实例.join();加入线程

泛型

  1. 使用泛型的原因
  • 把对象丢进集合当中,取出对象时会被当成Object类
  1. 使用方法

  • 在集合接口,类后增加尖括号,尖括号后放一个数据类型

  • 1.7之后可以不用带完整的泛型信息

  1. 泛型声明
  2. 子类派生
  • 子类不能使用类型形参

集合

比较规则

  • Comparable实现了该接口(重写方法compareTo(Object o))的类具备比较性,能将此类的实例放入有序集合当中并对集合中的对象进行排序比较的即为调用该方法的元素与Object
  • ==为没有自然顺序的对象collection提供排序==实现了Comparator接口(重写方法compare)的类是一个比较器,将泛型(规定的类型)传入比较器当中.使用时,将比较器作为集合的构造函数当中的形参传入到构造器当中,即定义了集合当中元素的比较方法==此比较器还可以作为参数传递到Arrays.sort()方法中,作为对象的比较规则==
  1. 集合类:Collection,Map
  2. Collection:
  • Queue:类似于List
  • Set:无法记住添加这个元素的顺序，只能通过元素本身来访问
  • List: 记住每次添加元素的顺序,直接通过索引访问
  • Iterator iterator();遍历对象
  • toArray(); 把集合转化为一个数组
  1. Map:key-value 根据key访问value
  2. Collection中的重要方法
  • ==所有实现了Iterable接口的类都可以使用迭代器,类用iterator()返回一个迭代器==
  • add(),remove(),clear()，size()等
  • Object[] toArray();把集合变成对应的数组，集合元素变成数组元素
  1. Set
  • Set中不允许添加重复元素
  • TreeSet处于排序状态
  • HashSet访问速度快
  • LinkedHashSet按元素添加的顺序访问，以链表维护内部顺序（==链表插入的速度快，数组查找的速度快==）

接口

1.基本语法格式

==一个类可以实现多个接口，但是只能有一个直接父类==

  1. 接口的实现

==方法名，返回值类型，参数个数及类型必须与接口中的完全一致，必须实现接口中的所有方法==

继承

  • ==重写== 子类方法名与父类方法名相同
  • 调用父类的构造方法
  • 只能放在子类构造器的第一行

  • 构造器被private修饰那就不能拥有子类了,子类无法被实例化,所以用super()调用构造器时不用考虑父类的构造器的封装修饰符

  • 操作被父类隐藏的成员变量和被重写的成员方法(利用super关键字要考虑父类属性封装类型)

  • 被定义为final的方法不能被重写

封装

  • private 同一个类当中
  • default 同一个包中
  • protected 子类中
  • public 全局
  • 访问范围 public>protected>default>private
  • 即使在该类的对象在此类外访问自身的private属性或者方法也是不允许的

枚举类

类的对象有限而且固定

  1. 枚举类的特性
  • enum关键字的地位与class,interface的地位相同
  • 默认继承java.lang.Enum
  • enum修饰:非abstract的枚举类用==final==修饰
  • 构造器默认修饰符为==private==
  • 所有实例必须在枚举类第一行显式列出，否则永远不会产生实例，实例的修饰符为==public static final==
  1. 枚举类的默认方法
  • values()返回该枚举类的所有实例
  • 使用实例时用EnumClass.variable
  • Enum类实现Comparable接口,有方法compareTo()(关于compareTo方法的复写详见API文档)
  • EnumClass.variable.ordinal()返回枚举值的索引
  1. 枚举类对象的创建

4.枚举类的实例类似于==匿名内部类==

  • ==枚举类当中的实例实际上就是匿名内部类，需要实现枚举类当中的抽象方法,枚举变量名实际上就是给匿名内部类”起名”==

  • //内部类 实例化内部类对象，外部类通过内部类对象调用方法

  • //匿名内部类 常用来实例接口

事件处理

  1. 事件模型:三个对象:事件源.事件,监听程序
  • 事件源:例如:按钮
  • 产生事件对象如:ActionEvent的对象
  • 编写监听器(实现特定类型接口),捕获事件对象,如:ActionEvent类对应的是ActionListener接口,实现接口当中所有的方法
  • 事件源加载监听器对象
  • 所有事件源的共有方法addXXXListener(),removeXXXListener();

1.

若try发生异常,则抛出异常给java运行时环境,交给catch

2.

Exception与Error都是Throwable的子类

3.

java1.7可以用一个catch语句捕获多个异常

4.

异常对象的信息

5.

回收资源

6.

异常分类

1. Runtime异常
2. Checked异常
checkedException

设计♂哲学（没有完善的错误处理机制的代码不会被执行）

  1. 方法知道如何处理该异常（try）
  2. 方法不知道如何处理该异常（throws）
